Especificaciones

Layout

  • 240px sidebar fija
  • flex:1 área main
  • 280px right rail
  • Shell full-height dark
  • 1440px canvas referencia

Componentes

  • Tab switcher (4 tabs)
  • KPI grid — 4 cards
  • Equity/Balance chart comparativo
  • Heatmap calendario
  • Data table trades
  • Direction toggle LONG/SHORT
  • Progress bars (ok/warn/crit)
  • Donut discipline score

Comportamiento

  • Tab switcher JS sin recarga
  • Balance+Equity: ambas líneas visibles
  • Heatmap coloreado por P&L
  • Right rail siempre visible
  • SEM badge actualiza cada lunes

Resumen de Funcionalidad

/dashboard

¿Qué hace?

  • → Vista central del estado de trading: cuenta, performance y disciplina
  • → Tab Portfolio: balance/equity, heatmap calendario, últimos trades
  • → Tab Operador: equity curve, métricas de sesión, rachas win/loss
  • → Tab Disciplina: score de disciplina, cumplimiento de reglas, progreso
  • → Tab Playbook: performance por setup, win rate y avg R comparados
  • → Right rail siempre visible: top setups + formulario nuevo trade

Tabs

  • Portfolio — balance total, equity, heatmap de P&L mensual, lista de trades recientes
  • Operador — equity curve Balance vs Equity, stats por sesión (NY AM/PM, London, Asia), rachas
  • Disciplina — score global 0-100, cumplimiento de reglas, violaciones, progreso semanal
  • Playbook — tabla de setups ordenada por avg R, win rate por setup, trades por setup

Tab · Disciplina

Resumen de Funcionalidad

/dashboard?tab=disciplina

¿Qué hace?

  • → Muestra el score de disciplina de la semana actual (0–100)
  • → Alerta DO-NOT-TAKE cuando se cumplen condiciones de stop
  • → Composición semanal: plan seguido vs parcial vs off-plan
  • → Heatmap de las últimas 12 semanas por severidad de violación
  • → Distribución de outcomes por R (histograma -3R a +4R)
  • → Ranking de reglas más violadas con tendencia semana a semana

Flujos de usuario

  • 1. Trader ve alerta DO-NOT-TAKE → "Ver detalle" muestra condiciones exactas
  • 2. Score 78/100 → referencia rápida del estado semanal
  • 3. Heatmap → identifica patrones de días problemáticos
  • 4. Distribución R → valida que el sistema tiene expectativa positiva
  • 5. Violaciones → "Ver registro" navega a lista completa con contexto por trade
  • 6. Tendencias ↓ = mejora, ↑ = empeora semana vs semana anterior

Tab · Playbook

Resumen de Funcionalidad

/dashboard?tab=playbook

¿Qué hace?

  • → Muestra rendimiento de cada setup en los últimos 90 días
  • → Cada setup card tiene sparkline de equity, Win%, Avg R y R acumulado
  • → Identifica la mejor y peor sesión para cada setup
  • → Heatmap Setup × Sesión: win rate por contexto horario
  • → Monitorea cumplimiento del A+ Checklist ítem por ítem
  • → Permite filtrar entre setups activos, pausados y descartados

Flujos de usuario

  • 1. Trader ve grid de setups → identifica cuáles tienen Cum negativo
  • 2. Click en card → navega a /playbook con ese setup seleccionado
  • 3. Setup × Sesión → decide en qué sesión operar cada setup
  • 4. A+ Checklist → detecta qué condiciones se están salteando más
  • 5. Toggle "Pausados" → ve setups desactivados temporalmente
  • 6. Toggle "Descartados" → ve setups eliminados del plan
