Especificaciones

Layout

  • ·240px sidebar + flex main + 340px rail
  • ·Screen body: height 520px
  • ·Main: data table en card

Componentes

  • ·Data table: Regla (nombre+descripción), Tipo badge, Severidad badge, Violaciones mono, Estado badge
  • ·Filter tabs: Sistema/Personalizadas/DNT/Violadas
  • ·+ Nueva regla CTA
  • ·Tipos: DNT/LÍMITE/GESTIÓN/NOTICIAS/HORARIO
  • ·Severidad: CRÍTICA/ALTA/MEDIA

Comportamiento

  • ·Violadas tab filtra solo reglas con violaciones > 0
  • ·DNT tab muestra solo reglas de tipo DNT
  • ·CRÍTICA reglas bloquean el trade automáticamente si se violan
  • ·Reglas activas evalúan antes de aceptar cada nuevo trade

Resumen de Funcionalidad

/reglas

¿Qué hace?

  • → Documenta y monitorea las reglas de trading del usuario
  • → Muestra cumplimiento histórico de cada regla en el período
  • → Detecta y contabiliza violaciones automáticamente al registrar trades
  • → Clasifica reglas por categoría (Risk, Proceso, Psicología)
  • → Permite activar/desactivar reglas individualmente

Flujos de usuario

  • 1. Usuario define sus reglas (max drawdown diario, no revenge, etc.)
  • 2. Sistema trackea cumplimiento al registrar cada trade
  • 3. Vista muestra % de cumplimiento por regla en el período
  • 4. Regla con violaciones → badge rojo + contador de incidencias
  • 5. "+" → modal para agregar nueva regla con categoría y condición
